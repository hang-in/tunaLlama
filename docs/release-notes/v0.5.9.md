# v0.5.9 - MCP spawn fix via wrapper script (2026-05-12)

## 한 줄

새 Claude Code 세션 / Codex 세션에서 `tunaLlama MCP Server: ✘ failed`
나오는 회귀 해결. `python` PATH 대신 wrapper script 가 `.venv/bin/python`
자동 fallback.

## 회귀 원인

v0.5.0 audit 의 must-fix 1 ("절대경로 → portable `python` + cwd 변수")
가 만든 회귀:

- `.mcp.json` 의 `command: "python"` 이 시스템 python 으로 spawn 됨.
- mise / pyenv / direnv 같은 shell hook 은 Claude Code / Codex 의 child
  process spawn 시 활성되지 않음.
- system python 으로는 fastmcp / anthropic SDK 등 deps 부재 → ImportError
  → server fail.
- 현재 활성 세션 (venv 활성 shell 에서 띄운 세션) 만 작동했고, 새 세션은
  바로 fail.

## 변경

### Wrapper script: `plugin/bin/tunallama-mcp`

shell wrapper 가 fallback 사슬을 수행:

```bash
if [ -x ".venv/bin/python" ]; then
  exec ".venv/bin/python" -m plugin.mcp_server "$@"
elif command -v python ...
```

`.mcp.json` 의 `command` 가 `${CLAUDE_PLUGIN_ROOT}/bin/tunallama-mcp` 로
변경. portable + venv robust 둘 다 확보.

### 테스트

- `test_mcp_json_registers_server`: command 가 wrapper 경로인지 검증
  (이전: `"python"`).
- `test_mcp_wrapper_script_exists_and_executable` (신규): wrapper 파일
  존재 + 실행권한 + shebang + `plugin.mcp_server` 포함 검증.

### Codex INSTALL.md

`codex mcp add` 명령도 wrapper 경유로 단순화:

```
# 이전
codex mcp add tunallama -- "$PLUGIN_ROOT/.venv/bin/python" -m plugin.mcp_server

# v0.5.9+
codex mcp add tunallama -- "$PLUGIN_ROOT/plugin/bin/tunallama-mcp"
```
