# v0.6.1 - Windows HTTP 데몬 안정화 (2026-07-04)

## 한 줄

v0.6.0 의 Windows HTTP 데몬 경로를 실사용까지 다듬은 패치 — 배치 파일 CRLF 버그
수정, 데몬을 **창 없이(hidden)** 실행, launcher Windows venv 분기 (이슈 #1).

## Fixed

### 배치 파일 CRLF (핵심 버그)
Windows 스크립트(`.cmd`/`.vbs`)가 LF-only 줄바꿈이면 **cmd.exe 가 다중 라인
`if/else` 블록을 오파싱**한다. 그 결과 venv python 탐지가 깨져 system python 으로
폴백 → `ModuleNotFoundError` / pydantic 버전 충돌로 데몬이 조용히 죽었다.
- `.gitattributes` 로 `*.cmd` / `*.bat` / `*.vbs` / `*.ps1` **eol=crlf** 강제.
- 기존 스크립트 CRLF 변환.

### 데몬을 창 없이 실행
로그온 자동시작 시 콘솔 창이 뜨던 문제.
- 스케줄 작업 → `wscript` → `tunallama-httpd.vbs`(hidden, SW_HIDE) → `cmd`(숨김) →
  `python.exe` 체인. 자식 프로세스가 숨겨진 콘솔을 상속 → 화면에 창 없음.
- 데몬 로그는 `~/.tunallama/httpd.log` 로 리다이렉트.
- `tunallama-win-setup.ps1` 의 스케줄 작업 action 을 wscript+VBS 로.

### launcher Windows venv 분기 (이슈 #1)
`plugin/bin/tunallama-mcp` (stdio bash launcher) 의 python 탐지에
`.venv/Scripts/python.exe` 분기 추가. Git Bash 경유 stdio spawn(Codex CLI 등)
에서 venv python 을 올바로 잡는다. GitHub 이슈 #1 종료.

## 참고
- mac/Linux 는 v0.6.0 그대로 (stdio 자동 spawn), 변경 없음.
- Windows 셋업/재셋업: `powershell -ExecutionPolicy Bypass -File plugin\bin\tunallama-win-setup.ps1`
