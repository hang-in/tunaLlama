## Codex CLI 실측

사용자 환경 Codex 0.128.0 의 plugin 상세 화면:

```
Skills            tunaLlama:delegate-to-ollama
Hooks             No plugin hooks.        ← 인식 안 됨
Apps              No plugin apps.
MCP Servers       tunallama
```

→ Codex 가 `plugin/hooks/hooks.json` 을 plugin install 시 자동 등록 안 함.
`codex features list` 의 `hooks  stable  true` 는 hooks **기능 자체**가
활성인 것이지 plugin 의 hooks.json 을 끌어쓴다는 의미가 아님.

### Codex 수동 등록 (v0.5.6 documentation 추가)

`~/.codex/config.toml` 에 직접 `[hooks]` 섹션 추가 시도 가능:

```toml
[[hooks.SessionStart]]
[[hooks.SessionStart.hooks]]
type = "command"
command = "python /path/to/tunaLlama/plugin/hooks/session_start.py"
```

단 schema 정확성은 Codex docs 확인 + 실측 필요. v0.6.0 후보.
