## v0.5.0 → v0.5.1

### Bug fix

**state.md auto-extract false positive (post-release bug)**

증상: Codex 세션에서 `tuna_generate_code` (구구단 함수) 위임 후 state.md
`Active Decisions` 에 다음 noise 가 자동 추가됨:
```
- (auto) usage (2-9)
- (auto) (2-9) ---")
```

원인: `extract.py` 의 정규식이 LLM 출력의 코드 블록 / docstring / 주석에서
`Default usage (2-9)` 같은 텍스트를 decision 으로 잘못 추출.

해결:
- `_strip_code_blocks()` - ``` ```...``` ``` 코드 펜스 안 텍스트 제거 후 추출
- `_is_meaningful()` - 알파벳/한글 4+ char 토큰 필수 + stopword (`usage` /
  `default` / `example` 등) 만으로 이뤄진 entry skip
- 4 regression test 추가 (코드 블록 skip / meaningless token skip /
  stopword only skip / 코드 블록 외부 entry 는 그대로 추출)

### 새 CLI 명령 - `tunallama state`

```bash
tunallama state show     # state.md 내용 출력
tunallama state path     # 파일 경로만 출력 (파이프 용)
tunallama state clean    # (auto) entry 삭제, manual/verified 보존
```

오염된 state.md 정리 가능. 4 단위 테스트.
