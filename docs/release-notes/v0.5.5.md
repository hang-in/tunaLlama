# v0.5.5 - SessionStart hook 출력 schema 정정

## 한 줄

context-mode 의 sessionstart.mjs 와 비교 → **SessionStart hook 의 정확한
JSON schema** 발견. flat `additionalContext` 가 아니라 nested
`hookSpecificOutput.hookEventName + additionalContext` 필요. v0.5.2 이후
3 번의 patch 모두 schema 부정확이 원인이었을 가능성.

## 발견 과정

v0.5.4 (duplicate hooks 제거 + matcher 필드) 적용 후에도 SENTINEL-v053
인식 안 됨. context-mode 의 `~/.claude/plugins/cache/context-mode/.../hooks/
sessionstart.mjs` 마지막 줄 비교:

```js
console.log(JSON.stringify({
  hookSpecificOutput: {
    hookEventName: "SessionStart",
    additionalContext,
  },
}));
```

우리 v0.5.4:
```python
sys.stdout.write(json.dumps({"additionalContext": ...}))  # ✗ flat
```

→ Claude Code 가 `hookSpecificOutput.hookEventName` 필드로 매칭 후
`additionalContext` 추출. flat 으로는 무시.

## 변경

`plugin/hooks/session_start.py`:

```diff
- payload = {"additionalContext": ...}
+ payload = {
+     "hookSpecificOutput": {
+         "hookEventName": "SessionStart",
+         "additionalContext": ...,
+     }
+ }
```

`tests/plugin/test_hook_session_start.py`: assertion 도 nested schema 검증.

→ SessionStart hook + state.md auto-prepend **완전 작동**.

추가 분석 필요.
