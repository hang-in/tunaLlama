# v0.5.4 - plugin.json hooks 필드 중복 제거 + agents 인식 확인

## 한 줄

v0.5.3 install 후 Claude Code UI 에서 발견: `plugin.json` 의 `"hooks":
"./hooks/hooks.json"` 필드가 **자동 로드와 중복** 으로 에러 → 제거.
부수 효과로 **agents / skills auto-discovery 작동 확인**.

## 발견

```
1 error:
  Failed to load hooks from .../plugin/hooks/hooks.json:
  Duplicate hooks file detected: ./hooks/hooks.json resolves to
  already-loaded file. The standard hooks/hooks.json is loaded
  automatically, so manifest.hooks should only reference additional
  hook files.
```

Claude Code 가 `plugin/hooks/hooks.json` 을 **자동 로드**함. v0.5.3 에서
`plugin.json` 에 `"hooks": "./hooks/hooks.json"` 추가했더니 중복.

## 변경

```diff
 // plugin/.claude-plugin/plugin.json
-  "skills": "./skills/",
-  "hooks": "./hooks/hooks.json"
+  "skills": "./skills/"
```

## 부수 발견 - agents / skills auto-discovery 작동

Claude Code 의 plugin 관리 UI 에 다음 components 정상 인식:

```
Installed components:
  ● Agents: tuna-developer       ← Claude Code 가 .md frontmatter 인식
  ● Skills: delegate-to-ollama
  ● Hooks: SessionStart, PreToolUse
```

→ README 매트릭스 update:
- **Agents auto-discovery (Claude Code)**: ✓ (이전 미실측 → 확정)
- **Skills auto-load (Claude Code)**: ✓
- **Hooks 등록 (Claude Code)**: ✓ - 단 실 실행 여부는 v0.5.4 재검증 필요
  (v0.5.3 에서는 duplicate error 로 hooks 등록 실패 가능성)
